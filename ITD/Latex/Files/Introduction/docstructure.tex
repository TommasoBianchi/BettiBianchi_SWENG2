The DD is organized into 6 main sections:

\begin{enumerate}
\item {\textbf{\nameref{sect:introduction}}}: this section contains an overview on the purpose and the scope of this document, toghether with a glossary with the definitions of the main terms we will refer to in the followings.
\item {\textbf{\nameref{sect:requirements}}}: this section contains a table that lists which of the the requirements of the project, already presented in detail in the RASD document, have been implemented or not.
\item {\textbf{\nameref{sect:framework}}}: this section contains a deep analysis of the framework and language that we have used to develop the project and the reasons of this choices.
\item {\textbf{\nameref{sect:codestructure}}}: this section describes the general structure of the final implementation project; it shows all the folders with a brief summary of their contents.
\item {\textbf{\nameref{sect:testing}}}: this section describes which types of tests that have been made on the project.
\item {\textbf{\nameref{sect:installation}}}: this section contains a detailed guide to install our project on the main operative systems; please note that due to the short time the installation procedure has been tested and is guaranteed to work only on Ubuntu 16.04 Xenial, and thus we suggest to use it or another similar Linux distribution.
\item {\textbf{\nameref{sect:appendix}}}: this section contains some information about the work behind the drafting of this
document, such as the amount of time spent by each of the authors, the tools used to produce all
the material and a revision history that keeps track of all the modifications.
\end{enumerate}
